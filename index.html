Infraestructura Global y Fiabilidad

Regiones AWS

Una región es adecuada para los servicios, datos y aplicaciones de una empresa.

Factores a considerar:

  • Conformidad con los requisitos legales y de gestión de datos
  • Proximidad a los clientes
  • Servicios disponibles dentro de una región
  • Precios

Zonas de Disponibilidad

Una zona de disponibilidad es un centro de datos único o un grupo de centros de datos dentro de una región.

  • Ubicadas a decenas de kilómetros de distancia entre sí
  • Suficientemente cerca para tener baja latencia
  • Suficientemente separadas para reducir el riesgo de desastres

Ubicaciones Perimetrales

Una ubicación perimetral es un sitio que Amazon CloudFront utiliza para almacenar copias de tu contenido más cerca de tus clientes para una entrega más rápida.

Aprovisionamiento de Recursos AWS

Formas de interactuar con los servicios de AWS

AWS Management Console

Interfaz basada en la web para acceder y administrar los servicios de AWS. Incluye asistentes y flujos de trabajo automatizados.

AWS Command Line Interface (CLI)

Permite controlar varios servicios de AWS directamente desde la línea de comando. Disponible para Windows, macOS y Linux.

Kits de Desarrollo de Software (SDK)

Facilitan el uso de los servicios de AWS a través de una API diseñada para el lenguaje de programación o plataforma específica.

AWS Elastic Beanstalk

El usuario proporciona ajustes de código y configuración, y Elastic Beanstalk implementa los recursos necesarios para:

  • Ajuste de la capacidad
  • Equilibrio de carga
  • Escalado automático
  • Seguimiento del estado de las aplicaciones

AWS CloudFormation

Permite tratar la infraestructura como código. Se puede crear un entorno escribiendo líneas de código en lugar de utilizar la AWS Management Console.

Conectividad con AWS

Amazon Virtual Private Cloud (VPC)

Un servicio de redes que permite establecer límites en torno a los recursos de AWS. Permite aprovisionar una sección aislada de la nube de AWS.

Puerta de Enlace de Internet

Conexión entre una VPC e Internet. Permite que el tráfico público de Internet acceda a tu VPC.

Puerta de Enlace Privada Virtual

Permite que el tráfico de Internet protegido entre en la VPC. Establece una conexión VPN entre la VPC y una red privada.

AWS Direct Connect

Servicio que permite establecer una conexión privada dedicada entre un centro de datos y una VPC.

Subredes y Listas de Control de Acceso

Subredes Públicas

Contienen recursos a los que el público debe tener acceso, como el sitio web de una tienda en línea.

Subredes Privadas

Contienen recursos a los que solo se debe tener acceso a través de la red privada, como una base de datos.

Listas de Control de Acceso a la Red (ACL)

Un firewall virtual que controla el tráfico entrante y saliente a nivel de subred.

  • Filtrado sin estado: No recuerdan nada y comprueban los paquetes en cada sentido
  • ACL predeterminada: Permite todo el tráfico entrante y saliente
  • ACL personalizadas: Todo el tráfico se deniega hasta que se agregan reglas

Grupos de Seguridad

Características

  • Firewall virtual para instancias EC2
  • Filtrado con estado
  • Deniega todo el tráfico entrante por defecto
  • Permite todo el tráfico saliente por defecto

Redes Globales

Resolución DNS

La resolución DNS es el proceso de traducir un nombre de dominio a una dirección IP.

Amazon Route 53

Servicio web DNS que ofrece a los desarrolladores y las empresas una forma fiable de dirigir a los usuarios finales a aplicaciones de Internet alojadas en AWS.

Características principales:

  • Conecta peticiones de usuarios a infraestructura AWS
  • Puede dirigir usuarios a infraestructura fuera de AWS
  • Administra registros DNS de nombres de dominio
  • Permite registrar nuevos nombres de dominio

Servicios de Almacenamiento

Almacén de Instancias

Proporciona almacenamiento temporal a nivel de bloque para una instancia de Amazon EC2. Los datos se pierden cuando termina la instancia.

Amazon EBS

Proporciona volúmenes de almacenamiento a nivel de bloque que se pueden usar con instancias EC2. Los datos persisten independientemente de la instancia.

Amazon S3 - Clases de Almacenamiento

S3 Standard

Para datos de acceso frecuente

S3 Standard-IA

Para datos de acceso poco frecuente

S3 Glacier

Para archivado de datos

S3 Glacier Deep Archive

Para archivado a largo plazo

Amazon EFS

Sistema de archivos escalable que se utiliza con los servicios en la nube de AWS y los recursos en las instalaciones. Se escala automáticamente.

Servicios de Bases de Datos

Bases de Datos Relacionales

Amazon RDS

Servicio administrado que automatiza tareas como aprovisionamiento de hardware, configuración, aplicación de parches y copias de seguridad.

Motores disponibles:
  • Amazon Aurora
  • PostgreSQL
  • MySQL
  • MariaDB
  • Oracle Database
  • Microsoft SQL Server

Bases de Datos No Relacionales

Amazon DynamoDB

Servicio de base de datos de valor de clave sin servidor con rendimiento de milisegundos de un dígito a cualquier escala.

  • Sin servidor
  • Escalado automático
  • Alto rendimiento

Servicios Adicionales

Amazon Redshift

Servicio de data warehousing para análisis de big data

AWS DMS

Servicio de migración de bases de datos
