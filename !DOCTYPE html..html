🌍 Infraestructura Global y Fiabilidad

+

Selección de Región Adecuada

La elección de la región adecuada para los servicios, datos y aplicaciones de una empresa es crucial. Los factores principales a considerar incluyen:

  • Conformidad con requisitos legales y de gestión de datos: Algunas regulaciones requieren que los datos permanezcan en jurisdicciones específicas
  • Proximidad a los clientes: Reduce la latencia y mejora la experiencia del usuario
  • Servicios disponibles dentro de una región: No todos los servicios AWS están disponibles en todas las regiones
  • Precios: Los costos pueden variar significativamente entre regiones

Zonas de Disponibilidad

Una zona de disponibilidad es un centro de datos único o un grupo de centros de datos dentro de una región.

Características clave:
  • Ubicadas a decenas de kilómetros de distancia entre sí
  • Suficientemente cerca para tener baja latencia
  • Suficientemente separadas para reducir el riesgo de desastres múltiples
  • Conectadas por enlaces de red de alta velocidad

Ubicaciones Perimetrales (Edge Locations)

Una ubicación perimetral es un sitio que Amazon CloudFront utiliza para almacenar copias en caché de tu contenido más cerca de tus clientes para una entrega más rápida.

  • Reduce la latencia al servir contenido desde ubicaciones cercanas
  • Mejora la experiencia del usuario final
  • Distribuye la carga de trabajo globalmente

⚙️ Aprovisionamiento de Recursos AWS

+

AWS Management Console

Interfaz basada en la web para acceder y administrar los servicios de AWS.

Características principales:
  • Acceso rápido a servicios utilizados recientemente
  • Búsqueda de servicios por nombre, palabra clave o acrónimo
  • Asistentes y flujos de trabajo automatizados
  • Aplicación móvil disponible para monitoreo
  • Soporte para múltiples identidades simultáneas

AWS Command Line Interface (CLI)

Herramienta que permite controlar varios servicios de AWS directamente desde la línea de comando.

  • Multiplataforma: Disponible para Windows, macOS y Linux
  • Automatización: Permite automatizar acciones mediante scripts
  • Eficiencia: Ahorra tiempo en peticiones de API repetitivas
  • Ejemplos: Lanzar instancias EC2, configurar Auto Scaling

Kits de Desarrollo de Software (SDK)

Los SDK facilitan el uso de servicios AWS a través de APIs diseñadas para lenguajes específicos.

  • Lenguajes soportados: C++, Java, .NET, Python, JavaScript, PHP, Ruby
  • Integración: Permite usar servicios AWS con aplicaciones existentes
  • Documentación: Incluye documentación completa y código de muestra

AWS Elastic Beanstalk

Servicio que simplifica el despliegue de aplicaciones. El usuario proporciona código y configuración, y Elastic Beanstalk se encarga de:

  • Ajuste de capacidad: Gestiona automáticamente los recursos
  • Equilibrio de carga: Distribuye el tráfico entre instancias
  • Escalado automático: Ajusta recursos según la demanda
  • Seguimiento del estado: Monitorea la salud de aplicaciones

AWS CloudFormation

Permite tratar la infraestructura como código, creando entornos mediante plantillas.

Beneficios clave:
  • Infraestructura como código: Define recursos usando JSON o YAML
  • Aprovisionamiento repetible: Garantiza consistencia
  • Sin intervención manual: Elimina scripts personalizados
  • Rollback automático: Deshace cambios si detecta errores

🔗 Conectividad con AWS

+

Amazon Virtual Private Cloud (VPC)

Servicio de redes que permite establecer límites seguros alrededor de los recursos de AWS.

  • Sección aislada: Proporciona una porción aislada de la nube AWS
  • Red virtual definida: Control completo sobre el entorno de red
  • Organización en subredes: Agrupa recursos según necesidades
  • Comunicación interna: Las subredes pueden comunicarse entre sí

Puertas de Enlace

Puerta de Enlace de Internet

Conexión entre una VPC e Internet para permitir tráfico público. Sin una puerta de enlace de Internet, nadie puede acceder a los recursos de la VPC.

Puerta de Enlace Privada Virtual

Permite establecer una conexión VPN entre la VPC y una red privada, como un centro de datos en las instalaciones.

AWS Direct Connect

Establece una conexión privada dedicada entre un centro de datos y una VPC.

  • Reduce costes de red
  • Aumenta el ancho de banda
  • Conexión más estable y predecible

Subredes y Control de Acceso

Subredes: Secciones de una VPC para agrupar recursos.

  • Públicas: Recursos accesibles al público (sitios web)
  • Privadas: Recursos solo accesibles internamente (bases de datos)

Listas de Control de Acceso (ACL)

  • Firewall virtual a nivel de subred
  • Filtrado sin estado (no recuerdan peticiones anteriores)
  • ACL predeterminada permite todo el tráfico
  • ACL personalizadas deniegan todo por defecto

Grupos de Seguridad

  • Firewall virtual a nivel de instancia EC2
  • Filtrado con estado (recuerdan peticiones anteriores)
  • Deniega todo el tráfico entrante por defecto
  • Permite todo el tráfico saliente por defecto

Amazon Route 53

Servicio web DNS que conecta peticiones de usuarios a infraestructura AWS.

  • Resolución DNS confiable (traduce nombres de dominio a direcciones IP)
  • Gestión de registros DNS
  • Registro de nuevos dominios
  • Integración con CloudFront

💾 Servicios de Almacenamiento

+

Almacenes de Instancias y Amazon EBS

Almacén de Instancias

  • Almacenamiento temporal a nivel de bloque
  • Conectado físicamente al equipo de alojamiento
  • Misma vida útil que la instancia
  • Importante: Se pierden los datos al terminar la instancia

Amazon Elastic Block Store (EBS)

  • Volúmenes de almacenamiento persistentes
  • Los datos permanecen disponibles aunque se detenga la instancia
  • Se puede adjuntar a instancias EC2
  • Permite crear instantáneas para copias de seguridad

Instantáneas de EBS

Copia de seguridad progresiva:

  • Primera copia: Copia todos los datos
  • Copias posteriores: Solo los bloques que han cambiado
  • Más eficiente que las copias completas

Amazon Simple Storage Service (S3)

Almacenamiento de Objetos

Cada objeto consta de:

  • Datos: Imagen, vídeo, documento o cualquier archivo
  • Metadatos: Información sobre los datos
  • Clave: Identificador único del objeto

Características de Amazon S3

  • Almacena datos como objetos en buckets
  • Control de permisos para visibilidad y acceso
  • Control de versiones para seguimiento de cambios
  • Al modificar un archivo, se actualiza todo el objeto

Tipos de Almacenamiento S3

  • S3 Standard: Acceso frecuente
  • S3 Standard-Infrequent Access: Acceso poco frecuente
  • S3 One Zone-Infrequent Access: Una sola zona
  • S3 Intelligent-Tiering: Optimización automática
  • S3 Glacier: Archivado a largo plazo
  • S3 Glacier Deep Archive: Archivado más económico

Amazon Elastic File System (EFS)

Almacenamiento de Archivos

Múltiples clientes pueden acceder a datos almacenados en carpetas compartidas.

  • Sistema de archivos escalable
  • Se utiliza con servicios AWS y recursos en las instalaciones
  • Aumenta y se reduce automáticamente
  • Ideal cuando múltiples servicios necesitan acceder a los mismos datos

Comparación EBS vs EFS

Amazon EBS:
  • Almacena datos en una única zona de disponibilidad
  • Debe estar en la misma zona que la instancia EC2
Amazon EFS:
  • Servicio regional
  • Almacena datos en múltiples zonas de disponibilidad
  • Acceso simultáneo desde todas las zonas

🗄️ Servicios de Bases de Datos

+

Amazon Relational Database Service (RDS)

Servicio que permite operar bases de datos relacionales en la nube de AWS.

  • Servicio administrado que automatiza tareas
  • Aprovisionamiento de hardware automático
  • Configuración, parches y copias de seguridad automáticas
  • Integración con otros servicios AWS
  • Cifrado en reposo y en tránsito

Motores de Base de Datos RDS

  • Amazon Aurora
  • PostgreSQL
  • MySQL
  • MariaDB
  • Oracle Database
  • Microsoft SQL Server

Amazon Aurora

Base de datos relacional de clase empresarial.

  • Compatible con MySQL y PostgreSQL
  • Reduce costes de base de datos
  • Reduce operaciones innecesarias de E/S
  • Recursos fiables y disponibles

Amazon DynamoDB

Servicio de base de datos de valor de clave sin servidor.

Características:
  • Sin servidor: No requiere aprovisionar ni administrar servidores
  • Escalado automático: Se ajusta automáticamente a los cambios
  • Rendimiento: Milisegundos de un dígito a cualquier escala
  • Flexibilidad: Se pueden añadir/eliminar atributos en cualquier momento

Bases de Datos No Relacionales

Utilizan pares de valor de clave:

  • Elementos (claves): Organizan los datos
  • Atributos (valores): Características de los datos
  • No todos los elementos necesitan los mismos atributos

Servicios Adicionales de Bases de Datos

  • Amazon Redshift: Data warehousing para análisis de big data
  • AWS Database Migration Service (DMS): Migración de bases de datos
  • Amazon DocumentDB: Base de datos de documentos
  • Amazon Neptune: Base de datos de grafos
  • Amazon QLDB: Base de datos de libro mayor
  • Amazon Managed Blockchain: Blockchain administrado
  • Amazon ElastiCache: Caché en memoria
  • DynamoDB Accelerator: Acelerador para DynamoDB
